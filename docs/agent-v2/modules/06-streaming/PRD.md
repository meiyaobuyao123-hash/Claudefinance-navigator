# M06 流式输出 — 产品需求文档

> 版本：v1.0 | 优先级：P1 | 状态：📋 规划中 | 最后更新：2026-03-22

---

## 1. 问题陈述

当前明理是等待 API 完整返回后一次性显示文字，用户等待 3-8 秒看到空白，极容易误以为 App 崩溃。这是用户流失的核心原因之一。

---

## 2. 用户故事

**US-01**：作为用户，我发送消息后应该在 500ms 内看到第一个字出现，哪怕只是一两个字。

**US-02**：作为用户，我希望看到文字像打字机一样逐字显示，让我知道 AI 正在思考和回复。

---

## 3. 功能需求

### 3.1 技术方案

使用 `anthropic_sdk_dart` 包的 SSE（Server-Sent Events）流式接口，替换当前的 `http.post` 一次性请求。

### 3.2 用户体验规格

| 指标 | 目标 |
|------|------|
| 首字符延迟（TTFT） | < 500ms |
| 流式更新频率 | 每次 chunk 立即渲染 |
| 等待状态 | 显示三点动画（"明理正在思考…"） |
| 网络断开 | 流式中断时显示"网络异常，消息已截断"+ 重试按钮 |

### 3.3 Markdown 渲染

- 使用 `flutter_markdown` 包实时渲染 AI 回复
- 流式过程中支持增量渲染（每次 chunk 追加后重新渲染整个 widget）
- 支持：加粗、斜体、无序列表、有序列表、代码块、分隔线

---

## 4. 验收标准

| 标准 | 验收方式 |
|------|----------|
| TTFT < 500ms（WiFi 环境） | 计时测试 |
| Markdown 加粗/列表正常渲染 | 人工视觉测试 |
| 流式中断后显示截断提示 | 断网测试 |
| 重试按钮可重新发送消息 | 人工测试 |

---

## 5. 优先级与范围

**MVP（必须有）**：SSE 流式 + 首字符延迟 < 500ms + Markdown 渲染

**V1.1（可延后）**：流式过程中的 Tool Use 支持（需收集完整 tool block 再执行）

**不做**：语音播报（TTS）
