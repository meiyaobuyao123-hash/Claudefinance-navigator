# M01 冷启动引导 — 产品需求文档

> 版本：v1.0 | 优先级：P0 | 状态：📋 规划中 | 最后更新：2026-03-22

---

## 1. 问题陈述

新用户第一次打开明理，没有任何档案、没有持仓数据、工具也查不到个性化信息。此时明理和普通 ChatGPT 没有区别——而第一次体验决定了用户是否会成为留存用户。

**核心矛盾**：明理需要用户信息才能给出个性化建议，但通过对话收集信息摩擦大、耗时长（平均需要 5-8 轮对话才能完成摸底）。

---

## 2. 目标用户

- 首次使用明理的新用户
- 已有规划页测评数据但从未使用明理的用户
- 清除数据后重新开始的用户

---

## 3. 用户故事

**US-01**：作为新用户，我希望在30秒内就能让明理了解我的基本情况，这样第一次对话就能获得针对性建议，而不是泛泛的通用内容。

**US-02**：作为已完成规划页测评的用户，我希望明理自动读取我的测评结果，不需要再重复填写。

**US-03**：作为用户，我希望引导流程可以跳过，直接开始对话，不被强制打断。

---

## 4. 功能需求

### 4.1 触发条件

冷启动引导在以下情况触发：
- 用户首次打开明理（`UserProfile` 为 null）
- 用户档案超过 180 天未更新（提示用户更新）

若已有规划页测评数据（R1-R5 + 资产配置），则**跳过引导，直接注入**，仅提示"已读取你的资产规划数据"。

### 4.2 引导流程（3步，每步独立一屏）

**Step 1：资产量级**
```
你目前有多少可以投资的钱？
（这帮助我给出合适的产品门槛建议）

○ 50万以下
○ 50-200万
○ 200-500万
○ 500万以上
```

**Step 2：核心目标**（多选，最多2个）
```
你最主要的理财目标是？

□ 跑赢通胀，本金安全
□ 稳健增值（年化3-6%）
□ 积极增值，接受波动
□ 养老规划（10年以上）
□ 子女教育金
□ 财富传承
```

**Step 3：风险校准问题**（单题，最关键）
```
去年A股大跌20%，你会怎么做？

○ 立刻止损卖出，睡不着觉
○ 有点担心，但观望不动
○ 正常，长期持有不在意
○ 加仓，这是买入机会
```

### 4.3 引导结果处理

- 3步完成后，生成 `UserProfile` 并持久化
- 立即注入 system prompt，**不显示 loading**，直接进入明理对话
- 明理第一句话主动引用档案："根据你的情况——200万可投资资金、稳健增值目标……"
- 引导期间不调用任何 API，纯本地

### 4.4 跳过机制

- 每一步右上角有"跳过"按钮
- 跳过后进入通用模式，明理会在对话中自然收集信息
- 跳过记录保存，下次打开不再显示引导（避免反复打扰）

---

## 5. 验收标准

| 标准 | 验收方式 |
|------|----------|
| 3步引导总时长 < 30 秒 | 用户测试计时 |
| 规划页有测评数据时自动读取，不显示引导 | 自动化测试 |
| 引导完成后第一条明理消息包含用户档案中的具体数据 | 人工测试 |
| 跳过后不重复显示引导 | 自动化测试 |
| 离线状态下可完成引导（无网络依赖） | 测试 |

---

## 6. 优先级与范围

**MVP（必须有）**：3步引导流程 + 自动读取规划页数据 + 跳过机制

**V1.1（可延后）**：引导数据与服务器同步（换设备后恢复）

**不做**：强制完成引导（始终可跳过）
