# M04 对话阶段状态机 — 产品需求文档

> 版本：v1.0 | 优先级：P1 | 状态：📋 规划中 | 最后更新：2026-03-22

---

## 1. 问题陈述

当前明理对每条消息的处理方式完全相同——无论用户是刚打开 App 还是已经聊了20轮，都用同一套 prompt。这导致：

- 用户刚进来就被疯狂"建议轰炸"，没有先了解情况
- 已经收集够信息了，还在不停追问
- 对话方向飘散，没有清晰的推进感
- 无法触发正确的 Tool Use 时机

---

## 2. 用户故事

**US-01**：作为新用户，我希望明理先问我几个关键问题，再给我建议，而不是上来就给一堆方案。

**US-02**：作为老用户，我希望明理能记得我们聊到哪里了，不要每次都从头介绍自己。

**US-03**：作为正在考虑行动的用户，我希望明理给我具体的"下一步"，而不是继续分析。

---

## 3. 功能需求

### 3.1 四个对话阶段

```
探索阶段 (Exploring)
  → 触发：新会话开始，用户档案不完整
  → 目标：通过2-3个问题摸清用户情况
  → 明理行为：多问少说，每次最多1个问题

深化阶段 (Deepening)
  → 触发：用户已提供足够背景信息
  → 目标：深入分析某个主题，给出专业见解
  → 明理行为：长段分析，引入市场数据

行动阶段 (Actioning)
  → 触发：用户表示想做决策/有明确需求
  → 目标：给出可执行的具体步骤
  → 明理行为：分点列举，提供产品导航链接

复盘阶段 (Reviewing)
  → 触发：用户提到过去的决策/持仓结果
  → 目标：帮用户客观复盘，不评判对错
  → 明理行为：先认可用户的决策逻辑，再给数据
```

### 3.2 阶段转换规则

```
探索 → 深化：用户发送消息数 ≥ 3，或已有完整档案
深化 → 行动：用户消息包含行动关键词（"怎么买"/"下一步"/"我想"）
任意阶段 → 复盘：用户消息包含复盘关键词（"当时"/"之前"/"我买了"）
行动 → 深化：用户问了新的分析性问题
```

### 3.3 对话摘要（防 context rot）

- 每当对话超过 **20 轮**或 token 超过 **8000**，触发自动摘要
- 明理生成一段对话摘要替换历史消息（保留最近5条）
- 摘要包含：讨论过的主要议题、用户确认的信息、尚未解决的问题

---

## 4. 验收标准

| 标准 | 验收方式 |
|------|----------|
| 新用户前3轮，明理不主动给建议 | 人工测试 |
| 用户说"我想买"，明理进入行动阶段 | 关键词触发测试 |
| 超过20轮后，不出现"之前我们聊到…"的幻觉 | 对话压力测试 |
| 阶段变化记录在会话 state，不影响 system prompt 大小 | 单元测试 |

---

## 5. 优先级与范围

**MVP（必须有）**：四阶段状态机 + 关键词触发转换 + 对话阶段注入 Layer 5

**V1.1（可延后）**：AI 自主判断阶段（让模型在 tool_use 中声明阶段转换）

**不做**：多轮对话图（DAG）——当前场景复杂度不够，状态机足矣
