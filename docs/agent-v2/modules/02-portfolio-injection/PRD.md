# M02 持仓上下文注入 — 产品需求文档

> 版本：v1.0 | 优先级：P0 | 状态：📋 规划中 | 最后更新：2026-03-22

---

## 1. 问题陈述

用户在持仓追踪器中已记录了完整的基金、股票持仓数据，但每次和明理对话，AI 对这些数据一无所知——只能给出泛泛的配置建议，无法落地到"你手上这只基金要不要换"的具体操作。

**核心矛盾**：持仓数据已有，但 AI 看不到。

---

## 2. 目标用户

- 已在持仓追踪器录入基金/股票数据的用户
- 向明理咨询"我这样的配置合理吗"的用户
- 需要明理针对具体持仓给出建议的用户

---

## 3. 用户故事

**US-01**：作为用户，我希望问明理"我现在的配置合理吗"时，它能直接分析我的实际持仓，而不是让我重新描述一遍。

**US-02**：作为用户，我希望明理知道我的基金盈亏情况，给出止盈/补仓建议时有数据依据。

**US-03**：作为用户，我不希望我的完整持仓数据每次都消耗大量 token，只在相关时注入即可。

---

## 4. 功能需求

### 4.1 数据范围

注入以下持仓快照数据（仅摘要，不含历史流水）：

```
基金持仓（每条）：
- 基金名称 + 代码
- 持仓市值
- 累计收益率%

股票持仓（每条）：
- 股票名称 + 代码 + 市场
- 持仓市值
- 累计收益率%

汇总：
- 总持仓市值
- 总累计收益率%
- 各大类占比（基金 vs 股票）
```

### 4.2 注入策略

**混合触发**（非全量注入）：

| 场景 | 注入内容 |
|------|----------|
| 用户发送持仓相关问题（含"持仓"/"基金"/"股票"/"配置"等关键词） | 完整持仓快照（~100 token） |
| 用户发送通用理财问题 | 仅注入总资产规模摘要（~20 token） |
| 持仓为空 | 不注入，引导用户添加持仓 |

### 4.3 持仓快照格式

```
【持仓快照 · 实时】
基金（3支）：总市值 28.4万，收益 +8.2%
  - 易方达蓝筹精选(005827)：12.1万，+18.5%
  - 中欧价值发现(166002)：9.3万，-2.1%
  - 招商中证白酒(161725)：7.0万，+12.8%
股票（2支）：总市值 15.6万，收益 -3.2%
  - 贵州茅台(600519·A股)：10.2万，+5.1%
  - 腾讯控股(0700·港股)：5.4万，-15.7%
合计：44.0万，收益 +3.8%
```

### 4.4 触发时机

- 每次对话发送前，由 Prompt 构建层判断是否注入
- 注入内容包含「实时」标记，明理可告知用户数据来源
- 持仓数据超过 24 小时未刷新时，快照标记改为「昨日」

---

## 5. 验收标准

| 标准 | 验收方式 |
|------|----------|
| 用户问"我的配置合理吗"时，明理能列举具体基金名称 | 人工测试 |
| 持仓为空时，明理不凭空编造持仓数据 | 人工测试 |
| 持仓快照 token 数 < 150 | 计数验证 |
| 通用问题不注入完整持仓（节省 token） | 日志验证 |

---

## 6. 优先级与范围

**MVP（必须有）**：完整持仓快照注入 + 关键词触发判断

**V1.1（可延后）**：基于持仓自动生成开场白（"你的基金整体+3.8%，要帮你分析近期表现？"）

**不做**：历史交易流水注入（token 过大）
