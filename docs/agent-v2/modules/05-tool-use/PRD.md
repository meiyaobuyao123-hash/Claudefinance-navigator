# M05 Tool Use 混合触发架构 — 产品需求文档

> 版本：v1.0 | 优先级：P1 | 状态：📋 规划中 | 最后更新：2026-03-22

---

## 1. 问题陈述

当前明理无法主动查询实时数据——只能依赖 system prompt 中预先注入的行情快照。当用户问"现在沪深300多少点？"或"余额宝利率是多少？"时，明理要么用过时数据回答，要么承认自己不知道。

---

## 2. 目标用户

- 问实时行情的用户（"黄金现在多少？"）
- 需要产品对比的用户（"货基和短债哪个收益高？"）
- 想查看自己持仓具体数据的用户（"我的易方达蓝筹今天涨多少？"）

---

## 3. 用户故事

**US-01**：作为用户，我问"现在黄金多少钱"，明理应该给我实时价格，而不是上次刷新的数据。

**US-02**：作为用户，我说"帮我看下我的基金"，明理应该主动触发持仓查询，而不是让我说"请注入持仓数据"。

**US-03**：作为用户，我希望 Tool 调用对我透明——明理可以说"我来查一下"，但不需要给我看原始 JSON。

---

## 4. 功能需求

### 4.1 工具清单（MVP）

| 工具名 | 触发场景 | 数据来源 |
|--------|----------|---------|
| `get_market_rates` | 问实时行情（A股/港股/黄金/货基利率） | MarketRateService |
| `get_portfolio_summary` | 问持仓/配置情况 | 本地 Riverpod Provider |
| `get_fund_detail` | 问某支基金的详情 | 天天基金 API |
| `get_product_info` | 问某类产品的介绍（如"R3理财是什么"） | 本地产品数据库 |

### 4.2 触发策略（混合）

**规则触发**（关键词检测，客户端主动插入）：
- 用户消息含"黄金"/"A股"/"沪深300"/"行情" → 预先调用 `get_market_rates`
- 用户消息含"我的持仓"/"我的基金" → 预先调用 `get_portfolio_summary`

**AI 自主触发**（`tool_use` 模式）：
- 将工具定义传给 Claude，让 Claude 自主决定何时调用
- 用于规则无法覆盖的边界情况

**融合策略**：规则触发优先执行（结果注入 prompt），AI 触发作为补充。

### 4.3 用户可见的 Tool 交互

```
用户：黄金现在多少钱？

明理：（内部调用 get_market_rates）

明理（回复）：黄金 Au9999 现价 ¥628.50/g，较昨日上涨 +0.3%。
近期受美联储降息预期支撑……
```

用户看不到 JSON，只看到自然语言结果。

### 4.4 Tool 失败处理

- 网络超时（5秒）：明理用缓存数据 + 声明"数据为15分钟前"
- 数据源不可用：明理声明"当前无法获取实时数据，以下仅供参考"
- 不降级静默处理（用户应知道数据时效性）

---

## 5. 验收标准

| 标准 | 验收方式 |
|------|----------|
| 问"黄金多少钱"时，回复包含实时价格和时间戳 | 人工测试 |
| Tool 调用失败时，明理会说明数据来源和时效 | 故障注入测试 |
| Tool 结果不出现在用户气泡中（透明调用） | UI 审查 |
| 规则触发 < 100ms（本地判断） | 性能测试 |

---

## 6. 优先级与范围

**MVP（必须有）**：`get_market_rates` + `get_portfolio_summary` + 规则触发

**V1.1（可延后）**：`get_fund_detail` + AI 自主 tool_use 模式

**不做**：Tool 执行任何写操作（不自动下单、不修改持仓）
