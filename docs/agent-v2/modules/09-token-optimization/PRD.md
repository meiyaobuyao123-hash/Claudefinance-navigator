# M09 Token 优化 — 产品需求文档

> 版本：v1.0 | 优先级：P1 | 状态：📋 规划中 | 最后更新：2026-03-22

---

## 1. 问题陈述

每次明理对话都向 Claude API 发送完整的 system prompt + 完整对话历史，导致：
- API 成本随用户活跃度线性增长（无上限）
- 长对话后延迟增加（输入 token 越多，首字符延迟越长）
- 不必要的上下文注入降低模型信噪比

目标：**在不影响产品体验和 Agent 智能的前提下，将平均每次对话的 input token 降低 50% 以上。**

---

## 2. 优化方向总览

| 优化点 | 预估节省 | 难度 | 优先级 |
|--------|---------|------|--------|
| Prompt Caching（Claude 原生） | system prompt 费用 -90% | 低 | P0 |
| 对话历史滑动窗口 | history token -60% | 低 | P0 |
| 按需注入（持仓/市场数据） | context token -40% | 低 | P0（已设计，需完善） |
| System prompt 精简 | system token -30% | 中 | P1 |
| 话题相关性过滤市场数据层 | market token -70% | 中 | P1 |

---

## 3. 功能需求

### 3.1 Prompt Caching（最高优先级）

Anthropic 支持对 system prompt 使用 `cache_control: {"type": "ephemeral"}` 标记，缓存时效 5 分钟，命中后 system prompt 费用降低 90%。

**适用场景**：同一用户连续对话时，system prompt 基本不变（人格层固定，用户档案低频变化）。

**用户无感知**，纯 API 层改造。

### 3.2 对话历史滑动窗口

| 参数 | 值 | 说明 |
|------|----|------|
| 保留最近消息数 | 8 条 | 覆盖 99% 的上下文需求 |
| 超出部分处理 | 压缩为1条摘要 | M04 对话状态机已有摘要逻辑 |
| 触发条件 | 历史消息 > 8 条 OR 估算 token > 4000 | 以 token 为准更精确 |

超过 8 条后，将最旧的消息交给 AI 摘要（复用 M04 `ConversationSummarizer`），历史变为：`[摘要条] + [最新8条]`。

### 3.3 按需注入优化（持仓 + 市场数据）

**持仓层（Layer 4）**：
- 当前：每条消息都注入摘要（~20 token）
- 优化后：仅在关键词命中时注入完整快照（~150 token），**无关键词时完全不注入**（0 token）
- 预估节省：非持仓对话节省 20 token/次，约占 15% 的对话

**市场数据层（Layer 2）**：
- 当前：每次都注入今日行情（~200 token）
- 优化后：话题不相关时不注入，相关时注入
- 不注入场景：养老规划、子女教育金规划、风险偏好讨论等纯个人规划话题
- 预估节省：约 30% 的对话不需要实时行情

### 3.4 System Prompt 精简

对人格层（Layer 1）进行精简，目标从 ~500 token 压缩到 ~300 token：
- 删除冗余的例句（few-shot examples 改为更简洁的 1 个示例）
- 合并重复的限制说明
- 保留：角色定位 + 核心能力 + 禁止行为 + 1个对话示例

**不影响范围**：明理的回答风格、专业度、合规约束。

## 4. 预期效果

| 场景 | 优化前 input token | 优化后 input token | 节省 |
|------|-------------------|-------------------|------|
| 新用户第1条消息 | ~1200 | ~500 | -58% |
| 老用户第10条消息 | ~3500 | ~1200 | -66% |
| 老用户第20条消息（无摘要） | ~7000 | ~1500 | -79% |

---

## 5. 验收标准

| 标准 | 验收方式 |
|------|----------|
| Prompt Caching 命中时 API 日志显示 cache_read_input_tokens | API 响应日志 |
| 第10条对话 history token < 2000 | debug 日志计数 |
| 非持仓问题不注入持仓快照 | 单元测试 |
| 非行情问题不注入市场数据层 | 单元测试 |
| 精简后明理回答风格无明显变化 | 人工对比测试（10题） |

---

## 6. 优先级与范围

**MVP（必须有）**：Prompt Caching + 对话历史滑动窗口（最高 ROI，改动最小）

**V1.1（可延后）**：市场数据话题相关性过滤 + system prompt 精简

**不做**：动态裁剪 system prompt 内容（由 AI 判断哪层可删）——不可预测，风险高
