# M03 分层 Prompt 架构 — 产品需求文档

> 版本：v1.0 | 优先级：P0 | 状态：📋 规划中 | 最后更新：2026-03-22

---

## 1. 问题陈述

当前 `ai_chat_page.dart` 的系统提示词是一个 ~2000 token 的单体字符串，存在以下问题：

- 人格描述、用户信息、持仓数据混在一起，难以维护
- 无法按需更新某一层（如持仓更新但人格不变）
- token 浪费：每次对话都全量发送，即使持仓无关的对话也带上完整持仓
- 无法做精准的 token 监控和裁剪

## 3. 用户故事

**US-01（开发者）**：作为开发者，我希望系统提示词按层组织，每一层可以独立更新，不需要改整个字符串。

**US-02（运营）**：作为运营，我希望修改明理的人格描述时，不需要担心破坏持仓注入逻辑。

**US-03（产品）**：作为产品，我希望能看到每次对话的 token 分布，知道哪层消耗最多。

## 4. 功能需求

### 4.1 分层结构

```
system prompt = [
  Layer 1: 人格层        (~500 token) — 固定，启动时加载一次
  Layer 2: 市场数据层    (~200 token) — 15分钟 TTL，有缓存
  Layer 3: 用户档案层    (~300 token) — 用户完成引导后更新
  Layer 4: 持仓快照层    (~100 token) — 按需注入（关键词触发）
  Layer 5: 对话阶段层    (~100 token) — 根据当前对话状态注入
]
总计控制 < 1200 token
```

### 4.2 各层说明

**Layer 1 — 人格层（固定）**：
- 明理的角色定位、能力边界、沟通风格
- 合规免责声明（不推荐具体代码、不执行交易）
- 示例对话风格（几个 shot）

**Layer 2 — 市场数据层（定时刷新）**：
- 今日关键行情（A股/港股/美股/黄金/货基利率）
- 来自 `marketRatesProvider`，15分钟自动更新
- 离线时该层为空

**Layer 3 — 用户档案层（会话级）**：
- 来自 M01 冷启动引导结果
- 可投资资金量级、理财目标、风险偏好
- 180天内有效

**Layer 4 — 持仓快照层（按需）**：
- 来自 M02 持仓注入模块
- 仅在用户问题涉及持仓时注入
- 每次构建 system prompt 时动态判断

**Layer 5 — 对话阶段层（动态）**：
- 来自 M04 对话状态机
- 当前阶段（探索/深化/行动/复盘）对应的提示策略
- 控制明理当前的对话目标

### 4.3 Token 预算管理

| 层 | 目标上限 | 超出处理 |
|----|---------|---------|
| 人格层 | 500 token | 警告日志（固定，不裁剪） |
| 市场数据层 | 200 token | 只保留最核心的5个数据点 |
| 用户档案层 | 300 token | 截断（档案字段优先级排序） |
| 持仓快照层 | 150 token | 只保留持仓前8支 |
| 对话阶段层 | 100 token | 固定简短 |
| **总计** | **1200 token** | 超出时按层优先级裁剪（4→2→3→5，1固定不动） |

---

## 5. 验收标准

| 标准 | 验收方式 |
|------|----------|
| 每次对话 system prompt token 数 < 1200 | 日志监控 |
| 修改人格层不影响其他层 | 代码审查 |
| 持仓快照层可独立启用/禁用 | 单元测试 |
| 市场数据层缓存 15 分钟，不重复请求 | 单元测试 |

---

## 6. 优先级与范围

**MVP（必须有）**：分层构建器 + 各层独立可替换 + 总 token 控制

**V1.1（可延后）**：token 使用数据上报到服务器（用于成本分析）

**不做**：动态向 LLM 询问"哪层最重要"（自动裁剪方案过于复杂）
