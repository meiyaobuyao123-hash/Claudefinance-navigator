# M08 评估与反馈系统 — 产品需求文档

> 版本：v1.0 | 优先级：P2 | 状态：📋 规划中 | 最后更新：2026-03-22

---

## 1. 问题陈述

目前不知道明理在哪里表现差——哪类问题回答质量低、哪条回复让用户不满意。没有数据就没有改进方向。

---

## 2. 用户故事

**US-01**：作为用户，当明理给出的回答有帮助时，我可以快速点赞，强化这类回答。

**US-02**：作为用户，当明理的回答没有帮助时，我可以点踩并选择原因，帮助改进。

**US-03（运营）**：作为运营，我希望看到每天差评最多的问题类型，以便优化 prompt。

---

## 3. 功能需求

### 3.1 反馈 UI

每条明理回复气泡右下角显示：
```
[👍]  [👎]
```

点击后：
- 👍：立即变为实心填充，不再追问
- 👎：弹出选择框（可选1个原因）：
  - "回答不准确"
  - "没有解决我的问题"
  - "建议太泛泛"
  - "其他"

### 3.2 数据上报

```json
{
  "session_id": "xxx",
  "message_id": "xxx",
  "user_question": "我应该买什么基金",
  "ai_response_preview": "根据你的情况...(前50字)",
  "rating": "thumbs_down",
  "reason": "建议太泛泛",
  "conversation_stage": "deepening",
  "timestamp": "2026-03-22T10:30:00Z",
  "device_id": "xxx"
}
```

上报到：`POST http://43.156.207.26/api/finance/feedback`

### 3.3 本地统计（无需服务器的基础指标）

- 本次会话满意度：本次会话中👍/(👍+👎)
- 提示用户："你的反馈帮助明理变得更好"

---

## 4. 验收标准

| 标准 | 验收方式 |
|------|----------|
| 每条 AI 回复有点赞/点踩按钮 | UI 审查 |
| 点踩后弹出原因选择 | 人工测试 |
| 反馈数据上报到服务器 | 服务器日志验证 |
| 弱网下反馈上报失败不影响对话 | 断网测试 |

---

## 5. 优先级与范围

**MVP（必须有）**：👍/👎 UI + 服务端上报

**V1.1（可延后）**：运营后台查看差评分布（需要前端管理界面）

**不做**：自动基于反馈微调 prompt（需要足够数据量才有意义）
