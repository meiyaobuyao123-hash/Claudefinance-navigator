# PRD — 规划模块（Planning）

> 版本：V1.0
> 状态：**已实现，功能完整**
> 最后更新：2026-03-23
> 对应代码：`lib/features/planning/presentation/pages/planning_page.dart`（2044 行）

---

## 1. 问题陈述

中国大陆中高净值用户（50 万–1000 万人民币）缺乏系统性的资产配置自检工具。他们通常将资产随机分布在活期、理财、股票等品类中，却没有客观标准衡量当前配置是否健康、是否匹配自身风险承受能力。

规划模块的核心解法：让用户自行填入各资产类别的占比 → 系统基于用户的个人画像（年龄、家庭负担、收入稳定性、资产规模）生成个性化评估阈值 → 对配置进行健康打分并逐条给出风险提示 → 提供标准化优化建议配置 → 若已完成风险问卷，还可展示当前配置与 R1–R5 建议配置之间的偏离度对比。

---

## 2. 目标用户 + 用户故事

### 目标用户
- 净资产 50 万–1000 万人民币的中国大陆个人投资者
- 有一定投资经验、持有多类资产（存款 + 股票 + 基金等）
- 希望以低门槛方式对自己的整体配置做一次系统性自检

### 用户故事

| # | 用户故事 |
|---|---------|
| US-01 | 作为用户，我希望输入我各类资产的占比，系统给我一个健康评分，让我知道配置是否合理。 |
| US-02 | 作为用户，我希望填写年龄、家庭负担等个人情况，让评估标准符合我的实际处境，而不是一刀切。 |
| US-03 | 作为用户，我希望完成一个风险测评问卷，得到 R1–R5 风险等级，并看到我的配置与该等级建议配置的偏差。 |
| US-04 | 作为用户，我希望展开某类资产（如增值资产）后，能细化填写 A 股、港股、美股的占比，获得更精细的分析。 |
| US-05 | 作为用户，我不确定怎么填写配置时，希望一键进入 AI 顾问（明理）寻求帮助。 |
| US-06 | 作为用户，我希望评分结果和问卷答案在下次打开 App 时仍然保留，不需要重填。 |

## 3. 功能需求

### 3.1 页面整体结构

`PlanningPage` 是一个单页 `Scaffold`，使用 `CustomScrollView` + `SliverToBoxAdapter` 垂直堆叠以下区块，从上到下依次为：

```
Header（标题区）
ProfileStrip（个人情况条）
QuizEntry（风险问卷入口条）
EvaluatorCard（当前配置 + 评估按钮）
ResultCard（评估结果，仅在点击评估后显示）
AiEntry（问明理入口）
```

---

### 3.2 区块详述

#### 3.2.1 Header（标题区）
- 显示固定大字标题"资产配置"（fontSize 30，FontWeight.w700）
- 副标题："填入各类占比 · 标准已基于你的情况个性化"

#### 3.2.2 ProfileStrip（个人情况条）
- 单行可点击条，显示当前用户风险等级标签（稳健保守 / 均衡平衡 / 积极进取）及风险容量得分（x/85）
- 点击后弹出 `_ProfileSheet`（Modal BottomSheet）

**_ProfileSheet（个人情况弹窗）**：
- 4 个选项组，全部为 Wrap 多选标签样式（类似 pill button，选中高亮主色）：

  | 维度 | 选项 | 影响 |
  |------|------|------|
  | 年龄段 | < 30岁 / 30–40岁 / 40–50岁 / 50–60岁 / > 60岁 | 年龄越大，风险评分越低 |
  | 家庭负担 | 轻（单身/无贷）/ 中（有贷或孩子）/ 重（贷款+孩子+赡养） | 负担越重，风险评分越低 |
  | 收入稳定性 | 稳定薪资 / 创业/自由职业 | 稳定收入 +15 分 |
  | 总资产规模 | < 100万 / 100–300万 / 300–1000万 / > 1000万 | 资产越多，风险评分越高 |

- 底部实时展示"风险容量 x/85 · [等级] | 评估标准已个性化调整"
- 每次修改后立即回调到主页（`_profile` 状态更新，`_hasEvaluated` 重置为 false）

**风险容量评分算法（满分 85 分）**：
```
年龄得分：[35, 28, 20, 10, 3]（对应 <30 / 30–40 / 40–50 / 50–60 / >60）
家庭负担：[20, 12, 3]（对应 轻 / 中 / 重）
收入稳定：稳定 +15，不稳定 +0
资产规模：[0, 5, 10, 15]（对应 <100万 / 100–300万 / 300–1000万 / >1000万）
```

**风险等级映射**：
- 总分 ≥ 60 → 积极进取（R5 色系）
- 总分 35–59 → 均衡平衡（R3 色系）
- 总分 < 35 → 稳健保守（R1 色系）

**个性化评估阈值（`_Thresholds`）**：

| 风险等级 | 流动资产（min–max）| 稳健资产（min）| 高风险上限 | 高弹性单项上限 |
|---------|-----------------|--------------|----------|-------------|
| 稳健保守 | 15%–35% | 35% | 35% | 8% |
| 均衡平衡 | 10%–30% | 20% | 55% | 18% |
| 积极进取 | 8%–25% | 12% | 68% | 28% |

#### 3.2.3 QuizEntry（风险问卷入口条）
- 未完成时：显示"完成风险测评（获取配置建议）"，背景为默认 surface 色
- 已完成时：显示"风险测评已完成" + R等级标签（如 R3 平衡），背景为主色调淡底
- 右侧操作文字：未完成显示"开始"，已完成显示"重测"
- 点击后弹出 `_RiskQuizSheet`

**_RiskQuizSheet（风险问卷弹窗）**：
- 5 道单选题，选项 A/B/C/D 各对应 0–3 分
- 题目内容（已在代码中固化）：

  | # | 题目 |
  |---|------|
  | Q1 | 投资组合一个月内下跌 20%，您会怎么做？ |
  | Q2 | 您计划的主要投资期限是多久？ |
  | Q3 | 月收入中储蓄/投资的比例？ |
  | Q4 | 对哪类产品最感兴趣？ |
  | Q5 | 收益翻倍但可能亏损 50% 的投资，您会？ |

- 右上角实时显示答题进度（已答 / 总题数）
- 底部实时显示当前得分和风险等级预览（全部作答后激活"查看结果"按钮）
- 提交后：调用 `onCompleted` 回调 → 主页计算 `_quizRLevel`（1–5）→ 持久化到 SharedPreferences（key: `quiz_answers`）

**风险等级换算（总分 0–15）**：
```
0–3  → R1 保守
4–6  → R2 稳健
7–9  → R3 平衡
10–12 → R4 成长
13–15 → R5 进取
```

**持久化**：答案列表以字符串数组形式存储到 `SharedPreferences`（key: `quiz_answers`），App 启动时自动恢复。

#### 3.2.4 EvaluatorCard（当前配置卡片）

包含以下子元素：

**标题行**：
- 左侧："当前配置"（fontSize 16）
- 右侧：`_TotalBadge`：合计值 + 颜色状态（合计 = 100% 显示绿色"合计 100%"，否则红色）

**_AllocationBar（彩色分配条）**：
- 横向彩色进度条，按各类资产占比比例切分，使用 `Expanded(flex: ...)` 实现
- 5 种固定颜色：流动资产绿 / 稳健资产蓝 / 增值资产橙 / 另类资产紫 / 高弹性资产红

**5 大 L1 资产行（`_buildAssetSection`）**：

每行包含：
- 左侧彩色圆点 + 资产名称 + 描述副标题
- 可选"细分"按钮（流动资产无此按钮）：点击展开/收起 L2 子维度
- 右侧百分比输入框（宽 72，数字键盘，colorized）

| 类别 | 描述 | 是否支持细分 | 默认值 |
|------|------|------------|--------|
| 流动资产 | 活期 · 货币基金 · 随时可取 | 否 | 15% |
| 稳健资产 | 国债 · 定存 · 银行理财 | 是 | 40% |
| 增值资产 | 股票 · 基金 · ETF | 是 | 30% |
| 另类资产 | 黄金 · 港险 · REITs | 是 | 10% |
| 高弹性资产 | 可转债 · BTC+ETH · 其他加密 | 是 | 5% |

**L2 细分配置区（`_buildSubSection`，懒加载）**：

展开时，子类按父类值等比初始化（按 `defaultRatios` 分配，最后一项取余量避免舍入误差），显示在父类行正下方（缩进 20）。

| 父类 | 子类（3项） | 子类默认比例 |
|------|-----------|------------|
| 稳健资产 | 定期存款/大额存单 / 国债 / 银行理财/债基 | 50% / 25% / 25% |
| 增值资产 | A股 / 港股 / 美股 | 67% / 17% / 16% |
| 另类资产 | 黄金 / 港险/储蓄险 / REITs | 50% / 30% / 20% |
| 高弹性资产 | 可转债 / BTC+ETH / 其他加密 | 60% / 30% / 10% |

子类展开区右上角显示"合计 x% / 应为 y%"，颜色根据是否匹配而变化（绿/红）。

**评估按钮**：
- 合计等于 100% 时可点击，点击后 `_hasEvaluated = true` 触发 `_evaluate()`
- 合计不等于 100% 时按钮置灰并提示"各项合计须等于 100%（当前 x%）"

#### 3.2.5 ResultCard（评估结果卡片）

仅在 `_isValid && _hasEvaluated` 时渲染，包含：

**配置健康评分区**：
- 左侧：大字评分（fontSize 52）+ 风险类型标签（稳健保守型 / 均衡配置型 / 积极进取型）+ 用户画像标签（"基于「x」画像"）
- 右侧：76×76 圆形进度条，中心显示"优"（≥80）/ "良"（≥60）/ "待改善"（<60）

**评分颜色**：
- ≥ 80 → AppColors.success（绿）
- ≥ 60 → AppColors.warning（黄）
- < 60 → AppColors.error（红）

**风险类型判断**：
- 流动 + 稳健 > 70% → "稳健保守型"
- 增值 + 高弹性 > 55% → "积极进取型"
- 其他 → "均衡配置型"

**L1 风险提示（`_IssueRow`）**：
- 使用 🔴 🟡 🟢 三级标记（high / medium / low）
- 触发规则（阈值因 `_UserProfile.riskLevel` 而异）：

  | 规则 | 扣分 | 级别 |
  |------|------|------|
  | 流动资产 < minLiquid | -25 | high |
  | 流动资产 > maxLiquid | -10 | low |
  | 稳健资产 < minStable | -15 | medium |
  | 增值+高弹性 > maxHighRisk | -20 | high |
  | 高弹性 > maxSpec | -15 | high |
  | 另类资产 = 0% | -5 | low |

**L2 细分维度提示**（仅展开了子维度且已触发时显示）：

| 场景 | 扣分 | 级别 |
|------|------|------|
| 稳健：定存占比 >90% | -5 | low |
| 稳健：债基占比 >90% | -5 | medium |
| 增值：A股占增值 >80% | -10 | high |
| 增值：无港股+美股 | -5 | medium |
| 增值：美股占增值 >60% | -8 | medium |
| 高弹：其他加密占高弹 >50% | -10 | high |
| 高弹：加密合计占高弹 >80% | -8 | medium |
| 高弹：有加密但无可转债 | -5 | low |
| 另类：无黄金 | -3 | low |
| 另类：无港险 | -3 | low |

**偏离度对比区（`_buildDeviationSection`）**：
- 仅在 `_quizCompleted && _quizRLevel > 0` 时显示
- 标题："与 Rx [等级名] 建议配置的偏离"
- 对每个 L1 资产类别显示：
  - 当前值 → 建议值（文字）
  - 状态标签：偏高（>3%）/ 偏低（<-3%）/ 适中（±3% 以内）
  - 6 高宽进度条：蓝灰色背景 + 灰色表示建议值 + 彩色表示当前值（红/橙/绿）

R1–R5 推荐配置（硬编码）：

| 等级 | 流动 | 稳健 | 增值 | 另类 | 高弹性 |
|-----|------|------|------|------|-------|
| R1 | 25% | 55% | 15% | 4% | 1% |
| R2 | 20% | 45% | 25% | 7% | 3% |
| R3 | 15% | 30% | 35% | 12% | 8% |
| R4 | 10% | 18% | 45% | 15% | 12% |
| R5 | 8% | 10% | 50% | 15% | 17% |

**优化建议配置按钮**：
- 点击"查看优化建议配置"后，将 L1 配置一键重置为固定值：
  ```
  流动 15% / 稳健 45% / 增值 28% / 另类 8% / 高弹性 4%
  ```
  同步更新所有 TextEditingController 和已展开的 L2 子维度（重新等比初始化），并将 `_hasEvaluated` 置为 true。

### 3.5 评估逻辑完整流程

```
1. 用户修改任意 L1/L2 输入 → _hasEvaluated 重置为 false
2. 用户点击"个人情况" → 修改 _UserProfile → _hasEvaluated 重置
3. 用户点击"评估我的配置"（需合计 = 100%）→ _hasEvaluated = true
4. build() 检测到 _isValid && _hasEvaluated → 调用 _evaluate()
5. _evaluate() 返回 {score, style, l1Issues, l2Issues}
6. ResultCard 渲染评分 + 风险提示 + 偏离度对比（如问卷已完成）
```

## 4. 边界条件

| 条件 | 处理方式 |
|------|---------|
| L1 合计 ≠ 100% | 评估按钮置灰，不触发评估 |
| L2 合计 ≠ 父类值 | 子类合计显示红色提示，但不阻止 L1 评估（L2 评估逻辑会运行，只需展开状态为 true）|
| 输入非数字 | `double.tryParse` 返回 null，不更新状态 |
| 输入超出范围 | `.clamp(0, 100)` 限制 |
| SharedPreferences 读取失败 | try/catch 静默处理，_quizCompleted 保持 false |
| SharedPreferences 写入失败 | try/catch 静默处理 |
| L1 值为 0 时展开 L2 | 子类全部初始化为 0（等比后均为 0）|
| 问卷答案 saved 列表长度 ≠ 5 | 不恢复，_quizCompleted 保持 false |
| 风险容量评分极端情况（< 30岁 + 单身 + 稳定 + >1000万）| 最高 85 分，等级：积极进取 |

## 7. 不做的范围（V1.0 当前未实现）

- L1 配置不持久化：退出 App 后配置回到默认值 15/40/30/10/5
- L2 细分配置不持久化：退出 App 后展开状态和子类值清空
- 无金额输入模式：只支持百分比，不支持按实际金额填写
- 无历史记录：无法查看过去的配置快照
- 无配置对比：无法对比"优化前后"差异（只能一键覆盖）
- 无图表可视化：无饼图、雷达图等（只有彩色分配条）
- 无保存/分享功能：配置结果不能截图分享或导出
- AI 未读取规划配置：点击"问明理"跳转 AI 页时不携带当前配置数据，AI 不会自动基于规划页数据给建议
- 个人情况不持久化：退出 App 后恢复默认画像
- 风险问卷仅 5 题：无法细化到监管要求的完整 KYC 问卷
- 优化建议为固定值（硬编码）：不基于用户个人情况动态生成推荐配置

## 8. V1.1 规划（合理的下一步）

### P1（高优先级）
1. **持久化 L1 配置和个人情况**：使用 SharedPreferences 或 Hive 保存，避免用户重复填写
2. **AI 携带配置数据**：跳转 `/chat` 时传递当前 L1 配置 + 风险等级，让明理自动上下文感知
3. **动态优化建议**：`_applyOptimized` 当前为固定硬编码，应根据用户 `_quizRLevel` 和 `_profile.riskLevel` 综合生成个性化推荐

### P2（中优先级）
4. **金额模式**：支持用户切换"百分比输入"和"金额输入"（自动换算）
5. **配置历史快照**：每次点击评估保存一次记录，支持翻看历史
6. **饼图可视化**：在分配条下方增加交互式饼图

### P3（低优先级）
7. **L3 子维度**（如 A 股展开 → 宽基/行业/主动）
8. **对比两份配置**：支持"我的配置 vs 推荐配置"并排对比
9. **导出/分享**：生成配置健康评分截图
