# 决策日记 — 产品需求文档

> 版本：v1.0 | 优先级：P0（核心护城河）| 状态：✅ 已实现 | 最后更新：2026-03-22

---

## 1. 问题陈述

投资者在做理财决策时往往凭感觉，事后复盘时已记不清当时的理由和市场背景：
- "我当时为什么买这只基金？"
- "这笔大额存单锁定3年，当时利率是多少？现在亏了吗？"
- "我在高点买入，但当时我是怎么想的？"

**核心矛盾**：决策本身和市场结果之间缺乏闭环，用户无法从历史决策中学习。

---

## 2. 目标用户

- 持有 50万–1000万人民币，主动管理理财的用户
- 有记录习惯但没有系统工具的用户
- 想通过复盘改进自己决策质量的用户

---

## 3. 用户故事

**US-01**：作为用户，我在买入一笔大额存单时，想记录下当时的理由和市场利率水平，以便3年后到期时回顾当时的判断是否准确。

**US-02**：作为用户，我想在3个月后自动被提醒"你当时买入的那只A股ETF，现在市场涨跌了多少，你的判断对了吗？"

**US-03**：作为用户，我在持仓追踪器里加仓一只基金后，希望能顺便记录这次操作的理由，不必重新打开决策日记手动填。

**US-04**：作为用户，我希望能看到过去所有决策的列表，以及每条决策的复盘状态（待复盘/已复盘/正确/错误/中性）。

## 4. 功能需求

### 4.1 数据模型

#### DecisionRecord（决策记录）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | String | 时间戳字符串（`DateTime.now().millisecondsSinceEpoch.toString()`），唯一标识 |
| type | String | 决策类型（见 DecisionType）|
| productCategory | String | 产品类别（见类别列表）|
| amount | double | 涉及金额（元）|
| rationale | String | 用户填写的决策理由（必填）|
| expectation | String | 预期类型（见 DecisionExpectation）|
| linkedHoldingId | String? | 关联持仓 ID（从持仓操作触发时自动填入，可为空）|
| csi300AtDecision | double? | 决策时沪深300代理点位（自动采集，实际取 sz510300 ETF 价格作为代理）|
| moneyYieldAtDecision | double? | 决策时货基7日年化（自动采集，%）|
| createdAt | DateTime | 记录创建时间 |
| checkpoints | List\<DecisionCheckpoint\> | 复盘检查点列表（最多3个）|

#### DecisionType（决策类型）

| 值 | 展示文字 | 说明 |
|----|---------|------|
| buy | 买入 | 首次买入或追加 |
| sell | 卖出/赎回 | 卖出或赎回持仓 |
| rebalance | 调仓 | 调整仓位比例 |
| renew | 续期 | 到期续签（大额存单等固收产品）|
| pass_ | 放弃操作 | 考虑过但最终没有操作（记录"不操作的理由"）|

#### DecisionExpectation（决策预期）

| 值 | 展示文字 | 作用 |
|----|---------|------|
| rate_down | 预期利率下降 | 触发固收类复盘引擎（对比利率变化）|
| price_up | 预期价格上涨 | 触发权益类复盘引擎（对比沪深300）|
| risk_hedge | 规避风险 | 通用复盘逻辑 |
| other | 其他 | 通用复盘逻辑 |

#### DecisionCheckpoint（复盘检查点）

| 字段 | 类型 | 说明 |
|------|------|------|
| period | String | '3个月' / '6个月' / '1年' |
| date | DateTime | 实际执行复盘的时间 |
| csi300 | double? | 复盘时沪深300点位 |
| moneyYield | double? | 复盘时货基7日年化（%）|
| judgement | String | 自动生成的复盘文字描述 |
| verdict | String | 判决：correct / incorrect / neutral / pending |

#### 产品类别列表（13类）

货币基金 / 定期存款 / 大额存单 / 国债 / 银行理财 / 债券基金 / A股ETF / 主动基金 / 港股 / 美股ETF / 黄金 / 保险 / 其他

### 4.2 核心操作流程

#### 4.2.1 创建决策记录

**入口一：决策日记页 → 右上角「+」按钮**
1. 用户填写完整表单（决策类型/产品类别/金额/理由/预期）
2. 保存时系统自动采集市场快照（沪深300点位 + 货基7日年化）
3. 生成 UUID，写入 Hive 本地 + 腾讯云 PostgreSQL 双写
4. 返回列表页，新记录置顶

**入口二：持仓操作完成后弹出 DecisionPromptSheet**
- 触发时机：添加基金/股票、基金加仓/减持、股票增持/减持 操作成功后
- 预填内容：决策类型（根据操作自动映射）、产品类别（根据资产类别映射）、金额（从操作金额自动填入）、linkedHoldingId（当前持仓 ID）
- 支持传入 `categoryOptions` 参数过滤可选类别（如基金持仓仅展示基金相关类别）
- 默认预期：卖出类型默认 `other`，其余类型默认 `price_up`
- 用户只需填：理由 + 确认预期
- 支持跳过（不强制）

持仓操作 → 决策类型映射：
| 持仓操作 | 决策类型 |
|---------|---------|
| 添加基金/股票 | buy |
| 基金加仓/股票增持 | buy |
| 基金减持/股票减持 | sell |

#### 4.2.2 查看决策列表

- 按 createdAt 倒序排列（最新在顶）
- 每条卡片展示：类型徽章 + 产品类别 + 金额 + 创建日期 + 理由摘要 + 最新复盘状态
- 顶部显示「待复盘横幅」：统计当前 hasPendingReview=true 的数量（纯展示，当前无点击跳转行为）
- 空状态：显示引导文字「记录每一次决策，让复盘有据可查」

**卡片复盘状态标签展示规则**：

| 状态 | 展示文字 |
|------|---------|
| 无检查点 + hasPendingReview=true | 🔔 可以复盘了 |
| 无检查点 + hasPendingReview=false | ⏳ 复盘中 |
| 最新检查点 verdict=correct | ✅ 判断正确 |
| 最新检查点 verdict=incorrect | ❌ 判断有偏差 |
| 最新检查点 verdict=neutral | — 中性 |
| 全部3次复盘完成 | 显示历史，不显示「立即复盘」按钮 |

#### 4.2.3 执行复盘

**触发方式**：
- 用户手动点击卡片上的「立即复盘」按钮
- （未来）推送通知提醒（当前未实现）

**复盘流程**：
1. 调用实时行情服务获取当前沪深300点位和货基7日年化
2. 调用 `generateJudgement()` 纯函数生成复盘文字和 verdict
3. 生成 DecisionCheckpoint，追加到 record.checkpoints
4. 持久化（Hive + 服务器双写）
5. UI 刷新，显示复盘结果

**复盘检查点时间规则**：
- 第1次：createdAt + 90天（3个月）
- 第2次：createdAt + 180天（6个月）
- 第3次：createdAt + 365天（1年）
- 全部完成后不再提示复盘

#### 4.2.4 删除决策记录

- 卡片底部行显示 🗑️ 图标按钮（`Icons.delete_outline`），点击弹出确认弹窗
- 确认后：先从服务器删除（静默失败），再从 Hive 本地删除，刷新列表
- 不可恢复，无回收站

### 4.3 复盘判断引擎（generateJudgement）

纯函数逻辑，输入：DecisionRecord + 当前市场数据，输出：{text, verdict}

#### 分支一：固定收益类 + 预期利率下降 + 买入

适用类别：大额存单 / 定期存款 / 国债 / 银行理财

比对指标：货基7日年化（决策时 vs 复盘时）

| 利率变化 | verdict | 文字描述 |
|---------|---------|---------|
| 下降 > 0.05% | correct | "成功锁定了更高利率，每年多收益 X 元" |
| 上升 > 0.05% | incorrect | "利率不降反升，当时锁定时机偏早" |
| 变化 ≤ 0.05% | neutral | "利率变化不明显，判断基本中性" |
| 数据不可用 | neutral | "利率数据暂无法获取，建议手动对比" |

#### 分支二：权益类 + 预期价格上涨

适用类别：A股ETF / 主动基金 / 港股 / 美股ETF

比对指标：沪深300代理（sz510300 ETF 价格，决策时 vs 复盘时）

| 沪深300变化 | verdict | 文字描述 |
|------------|---------|---------|
| 上涨 > 3% | correct | "市场上涨约 X%，印证了你的判断" |
| 下跌 > 15% | incorrect | "市场下跌 X%，短期持仓需注意风险" |
| 下跌 3-15% | neutral | "市场下跌 X%，若长期持仓属正常波动" |
| 波动 ≤ 3% | neutral | "市场基本持平，方向尚不明确" |
| 数据不可用 | neutral | "暂无市场数据，无法自动对比" |

#### 分支三：通用逻辑（其余所有情况）

输出：提醒用户对照当时理由自主判断，verdict = neutral

### 4.4 数据存储与同步

**双写策略**：
- 本地：Hive（key = record.id，value = record.toJsonString()）
- 服务端：POST /api/finance/decisions（腾讯云 PostgreSQL decision_records 表）

**加载策略**：本地 Hive 优先（快速展示），然后异步拉取服务器数据替换（服务器为最终一致性来源）；服务器不可用时继续用本地数据

**服务端表结构**（decision_records）：
| 字段 | 类型 | 说明 |
|------|------|------|
| record_id | TEXT | PRIMARY KEY |
| device_id | TEXT | 设备 ID（用于隔离用户数据）|
| payload | TEXT | JSON 字符串（完整 DecisionRecord）|
| created_at | TIMESTAMPTZ | 记录时间 |

### 4.5 与其他模块的联动

| 联动方向 | 触发时机 | 行为 |
|---------|---------|------|
| 持仓追踪器 → 决策日记 | 添加/加仓/减持持仓操作成功后 | 弹出 DecisionPromptSheet，预填数据 |
| 持仓追踪器 → 决策日记 | 持仓操作菜单 | 「查看决策记录」入口，跳转 /decisions |
| 决策日记 → 市场行情 | 创建记录/执行复盘时 | 自动采集沪深300 + 货基7日年化 |
| 决策日记 → AI对话 | （规划中）向明理问"我这次决策对吗" | 携带决策上下文注入 prompt |

## 5. 边界条件

| 场景 | 处理方式 |
|------|---------|
| 市场行情采集失败 | 快照字段为 null，复盘时 verdict = neutral + 提示"数据不可用" |
| 服务器不可用 | 降级本地 Hive，联网后不自动同步（每次启动重新拉服务器最新数据）|
| 离线时执行复盘 | 获取不到实时数据，显示"当前离线，无法获取市场数据"，取消复盘 |
| 全部3个检查点都已复盘 | 不再显示「立即复盘」按钮，只显示历史复盘记录 |
| 删除有关联持仓的决策记录 | 仅删除记录，不影响持仓追踪器中的持仓数据 |
| 决策金额为 0 | 表单验证拦截，金额必须 > 0 |
| 理由为空 | 表单验证拦截，理由为必填项 |

## 7. 不做的范围

- **不做**：基于 AI 自动评分（目前用确定性规则，足够可预测）
- **不做**：复盘推送通知（flutter_local_notifications 已引入但未接入此模块）
- **不做**：决策记录的编辑（只能删除后重建，避免篡改历史）
- **不做**：多用户/家庭共享（当前用 device_id 隔离）
- **不做**：历史统计分析图表（V1.1 可延后）
- **不做**：导出 PDF / Excel
